# Настройка Telegram-бота для отчётов (по шагам)

## Шаг 1. Создать бота в Telegram

1. Откройте Telegram и найдите **@BotFather** (официальный бот для создания ботов).
2. Напишите ему команду: **`/newbot`**.
3. BotFather спросит **имя бота** (как он будет отображаться) — например: `Отчёт по инновациям в горнодобыче`.
4. Потом попросит **username** (должен заканчиваться на `bot`) — например: `mining_innovation_report_bot` или `yourname_report_bot`.
5. В ответ BotFather пришлёт **токен** вида:
   ```
   7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
   **Скопируйте этот токен** — это ваш **TELEGRAM_BOT_TOKEN**. Никому не показывайте и не публикуйте его.

---

## Шаг 2. Узнать свой Chat ID

Chat ID — это идентификатор вашего чата с ботом (или группы), куда бот будет присылать отчёты.

**Способ A (простой):**
1. Найдите в Telegram бота **@userinfobot**.
2. Напишите ему любое сообщение (например, «Привет»).
3. Он пришлёт вам информацию, в том числе **Id** — это ваш **Chat ID** (число, например `123456789`). Скопируйте его.

**Способ B (через своего бота):**
1. Напишите вашему новому боту (которого создали в шаге 1) любое сообщение, например «Привет».
2. Откройте в браузере (подставьте свой токен вместо `ВАШ_ТОКЕН`):
   ```
   https://api.telegram.org/botВАШ_ТОКЕН/getUpdates
   ```
3. В ответе в JSON найдите `"chat":{"id":123456789,...}` — число `id` и есть ваш Chat ID.

---

## Шаг 3. Добавить переменные в Railway

1. Зайдите на [railway.app](https://railway.app) и откройте ваш проект с агентом.
2. Выберите **сервис** (ваше приложение).
3. Вкладка **Variables** (Переменные).
4. Нажмите **Add Variable** (или **New Variable**) и добавьте по одной:

| Имя переменной | Значение | Пример |
|----------------|----------|--------|
| `TELEGRAM_BOT_TOKEN` | Токен из шага 1 | `7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` |
| `TELEGRAM_WEBHOOK_URL` | **Обязательно для Railway.** URL вашего приложения + `/telegram-webhook` | `https://vps-autonomous-agent-production.up.railway.app/telegram-webhook` |
| `TELEGRAM_REPORT_CHAT_IDS` | Ваш Chat ID из шага 2 | `123456789` |
| `TELEGRAM_ALLOWED_CHAT_IDS` | Тот же Chat ID (кто может писать боту) | `123456789` |

**Почему TELEGRAM_WEBHOOK_URL:** на Railway long polling часто не работает. Webhook надёжно доставляет сообщения на ваш сервер. Подставьте **свой** домен из Railway (Settings → Networking → Domain).

Если отчёты должны получать несколько людей — перечислите Chat ID через запятую без пробелов:
```
TELEGRAM_REPORT_CHAT_IDS=123456789,987654321
TELEGRAM_ALLOWED_CHAT_IDS=123456789,987654321
```

5. Сохраните переменные. Railway перезапустит сервис автоматически.

---

## Шаг 4. Проверить

1. Подождите 1–2 минуты после сохранения переменных (пока сервис перезапустится).
2. В Telegram откройте чат с вашим ботом.
3. Напишите команду **`/report`** или слово **`отчёт`**.
4. Бот должен ответить: «Формирую отчёт за последний день…» и через некоторое время прислать PDF-файл (если в БД уже есть данные за последний день).

Если бот не отвечает — посмотрите логи в Railway: **Deployments** → последний деплой → **View Logs**. Должна быть строка `[telegram] Бот запущен, команда /report — запрос отчёта`. Если её нет — проверьте, что **TELEGRAM_BOT_TOKEN** записан без лишних пробелов и кавычек.

---

## Краткий чеклист

- [ ] Узнан свой Chat ID (через @userinfobot или getUpdates).
- [ ] В Railway добавлены: `TELEGRAM_BOT_TOKEN`, `TELEGRAM_REPORT_CHAT_IDS`, `TELEGRAM_ALLOWED_CHAT_IDS`.

---

## Если бот не отвечает или отчёт не приходит

1. **Напишите боту `/start`**  
   Бот должен ответить текстом с вашим **Chat ID**. Если ответа нет — бот не получает сообщения (см. п. 4–5).

2. **Проверьте Chat ID**  
   Убедитесь, что в Railway в **TELEGRAM_REPORT_CHAT_IDS** и **TELEGRAM_ALLOWED_CHAT_IDS** указан **тот же** ID, что бот показал в ответ на `/start` (без пробелов, один номер или несколько через запятую).

3. **Проверьте токен**  
   В **TELEGRAM_BOT_TOKEN** должен быть токен от @BotFather без кавычек и пробелов в начале/конце.

4. **Логи в Railway**  
   Сервис → **Deployments** → последний деплой → **View Logs**. Должна быть строка:  
   `[telegram] Бот запущен. /start — подсказка...`  
   Если её нет — переменная **TELEGRAM_BOT_TOKEN** не подхватилась или сервис упал при старте бота. Проверьте переменные и перезапустите (Redeploy).

5. **На Railway обязательно задайте TELEGRAM_WEBHOOK_URL**  
   Long polling на Railway часто не работает. Добавьте переменную **TELEGRAM_WEBHOOK_URL** = `https://ВАШ-ДОМЕН.up.railway.app/telegram-webhook` (подставьте свой домен из Settings → Networking). Сохраните переменные и сделайте Redeploy. В логах должна появиться строка `[telegram] Webhook установлен`.

6. **Отчёт «Формирую…» пришёл, но PDF нет**  
   Подождите 1–2 минуты. Если PDF так и не пришёл — откройте логи Railway и посмотрите строки с `[done]` или ошибками по задаче отчёта: возможно, в БД ещё нет статей за последний день (сначала запустите задачу сбора `{"type":"collect"}` через webhook или дождитесь ночного сбора).
