# Сценарий: мониторинг производственной безопасности в горнодобыче

Задача: круглосуточный поиск в интернете данных о производственной безопасности в горнодобывающей отрасли, база данных, PDF-отчёт раз в сутки или по запросу, доставка в Telegram.

## Что умеет текущий агент

- Принимает текстовый запрос (prompt) и отправляет его в LLM (OpenAI).
- Ответ пишется только в логи (Railway/VPS). Нет выхода в интернет, PDF, Telegram, БД, расписания.

Поэтому **полный сценарий «поиск → БД → PDF → Telegram» текущей версией не реализован**. Ниже — что нужно добавить и как можно разбить на этапы.

## Что нужно для полной реализации

| Компонент | Назначение | Варианты |
|-----------|------------|----------|
| **Поиск в интернете** | Сбор данных о безопасности в горнодобыче | API поиска (Google Custom Search, Serper, Bing), парсинг выбранных сайтов (Puppeteer/Cheerio), RSS, новостные API |
| **База данных** | Хранение собранных материалов | SQLite (файл), PostgreSQL (Railway/VPS), или JSON-файлы для минимального варианта |
| **Формирование PDF** | Ежедневный отчёт | pdfkit, puppeteer (HTML→PDF), или шаблон (Markdown → PDF) |
| **Отправка в Telegram** | Раз в сутки + по запросу | Telegram Bot API: бот отправляет файл и/или текст; команда типа «Пришли отчёт» от вас |
| **Расписание** | «Раз в сутки» | Cron на VPS, или таймер внутри агента (setInterval), или n8n Schedule |
| **Принудительный запрос** | «По моему запросу» | Команда боту в Telegram («отчёт», «report») или webhook (curl / n8n), который запускает формирование и отправку отчёта |

## Варианты реализации

### Вариант A: Расширить этого агента

- Добавить в проект: модуль поиска (вызов API поиска), схему БД и запись результатов, генерацию PDF, Telegram-бота (отправка отчёта + обработка команды «пришли отчёт»).
- Запуск по расписанию: cron раз в день вызывает `POST /task` с задачей «сформировать и отправить отчёт», либо внутренний таймер в агенте.

### Вариант B: n8n как оркестратор

- **Поиск:** нода HTTP Request к API поиска или парсинг страниц.
- **БД:** нода для записи в БД (Postgres/SQLite через соответствующий нод или код).
- **PDF:** отдельный сервис или нода с кодом (pdfkit и т.п.).
- **Telegram:** нода Telegram — отправка сообщений/файлов, триггер по сообщению от вас.
- **Расписание:** триггер Schedule (раз в день) + ручной триггер или webhook для «по запросу».

### Вариант C: Гибрид

- Агент на Railway занимается только тем, что уже умеет (обработка текста, структурирование), или только поиском по API.
- n8n раз в сутки и по команде в Telegram запускает сценарий: поиск → сохранение в БД → генерация PDF → отправка в Telegram.

## Как отправить задачу текущему агенту (из Cursor)

Чтобы команда `npm run send-task "…"` работала:

1. В корне проекта создайте или отредактируйте `.env`.
2. Добавьте (подставьте свой домен и секрет из Railway):
   ```
   AGENT_URL=https://ваш-проект.up.railway.app
   WEBHOOK_SECRET=ваш_секрет
   ```
3. В терминале Cursor:
   ```bash
   npm run send-task "Ваш текст задачи"
   ```

Агент примет задачу и ответит в логах Railway (Deployments → View Logs). Для реализации полного сценария нужна доработка по одному из вариантов выше.
